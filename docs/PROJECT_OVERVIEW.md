# Tillers Staffing — Project Overview

Internal tool for managing student staffing operations: planning, contracts, hour tracking, and payroll handoff.

---

## Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 15 (TypeScript, App Router) |
| Database | PostgreSQL via Neon |
| ORM | Prisma |
| Auth | NextAuth.js (credentials) |
| File Storage | Vercel Blob |
| Email | Resend (later: WhatsApp via SIM) |
| Payroll | Nmbrs (future — design for sync now) |
| Deployment | Vercel |
| CAO | NBBU CAO |

---

## Roles

| Role | Description |
|---|---|
| `ADMIN` | Full access. Creates jobs, approves hours, manages contracts, sends invoices |
| `EMPLOYER` | Views their jobs/shifts, approves student hours |
| `STUDENT` | Fills in own profile, views schedule, clocks in/out, signs contracts |

Auth is intentionally kept simple for the first version.

## Key Business Rules

- All workers are EU nationals — no work permit tracking needed
- NBBU CAO applies to all student contracts
- Fase A lasts 52 worked weeks; admin decides whether to move student to Fase B after
- Employers have individual markup rates over student hourly rates
- Employers approve student hours; admin does final confirmation before payroll
- Payroll calculations, payslips, and payouts are handled by Nmbrs (future)
- GPS required for student clock-in
- WhatsApp notifications planned (future); email in first version

## Data Model Summary

```
User (ADMIN | EMPLOYER | STUDENT)
 ├── Student
 │    ├── Contract (fase A/B/C)
 │    ├── Shift (many)
 │    └── Payslip (many, generated by Nmbrs)
 └── Employer
      ├── Job (many)
      │    └── Shift (many, links Job ↔ Student)
      └── Invoice (future)
```

## Development Phases

### Phase 1 — Core (build first)
- Student & employer profiles
- Fase A contract generation + e-signing

- Hour approval flow (employer → admin)

### Phase 2 — Payroll Handoff
